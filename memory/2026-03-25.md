## 2:10 PM PST — CRITICAL SECURITY BUG FIXED

**Issue:** Data isolation bug allowing cross-tenant memory access. Different API keys could see each other's memories if they used the same agent_id.

**Root cause:**
- API endpoints received tenant auth but didn't call `set_tenant_context()` before DB operations
- `recall_fixed()` function hardcoded `tenant_id="00000000-0000-0000-0000-000000000000"` for ALL queries
- Recall queries filtered by `agent_id` only, not `tenant_id` — complete RLS bypass

**Fix applied:**
- Added `set_tenant_context(tenant["id"])` to all 37 authenticated API endpoints
- Updated `recall.py`: Added `tenant_id` parameter to all functions, passed through entire call chain
- Added `AND tenant_id = %s::UUID` to ALL memory queries (semantic search, high-importance, keyword search, corrections, handoffs)
- Updated recall cache key to include tenant_id (prevents cache poisoning)
- Fixed async extraction background thread (thread-local context doesn't carry over)

**Verification:**
- API restarted successfully
- Health check: Status OK, 854 memories, Redis connected
- All imports clean

**Impact:** Production security vulnerability patched. No known exploitation (pre-launch).
