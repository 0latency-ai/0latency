### Memory Product Refinements Identified (Echo's proposals)

**Recall quality:**
- Recency weighting in recall — time-weighted ranking ("2 hours ago" beats "3 weeks ago")
- Recall feedback loop — reinforce memories that get used in responses, demote ones that get surfaced but ignored. System learns what's useful, not just what's relevant. **This is the highest priority refinement.**

**Extraction quality:**
- Intent detection — distinguish "stated a fact" vs "brainstorming" vs "venting." Different memory types and decay rates.
- Conversation phase awareness — early turns = context-setting, middle = working, final = decisions/summaries. Weight extraction accordingly.

**Operational:**
- Memory health dashboard — total count, extraction rate, recall hit rate, contradiction flags, orphaned memories
- Audit trail — "why did you think X?" traces to exact conversation turn. Source attribution in schema but needs exposure to agent.

**Scale prep:**
- Memory partitioning by domain (PFL, SSU, personal, infrastructure) — matters at 500+ memories
- Ephemeral memory type with aggressive TTL — "I'm at Waterbar tonight" useful for 12 hours, not 12 months. Forgetting with intent.

## Gap Analysis #3 + Infrastructure Fixes (5:53 PM UTC)

### What Triggered This
Justin sent ~30 screenshots from March 6 (mem0 origin story). Previous-Thomas responded to EACH ONE individually — play-by-play commentary consuming 15K+ tokens. This triggered compaction. Post-compaction, Thomas couldn't properly rehydrate and lost coherence.

### 5 Failures Identified
1. **Screenshot-per-response pattern** — No batching logic. Each inbound image treated as independent prompt requiring full response. Should wait for all media, then respond once.
2. **Compaction destroyed working context** — Verbose screenshot responses pushed out important context (product work, pricing decisions, deployment checklist).
3. **Post-compaction rehydration incomplete** — HANDOFF.md was stale, MEMORY_CONTEXT.md surfaced generic memories, agent didn't trust/follow its own startup instructions.
4. **Blocking during long work runs** — Main session becomes unreachable during long tool chains. Justin can't interrupt or redirect.
5. **Naming confusion** — MEMORY_CONTEXT.md and MEMORY.md too similar. Confusing which is authoritative post-compaction.

### Fixes Implemented
1. **Media Batching Rule** added to AGENTS.md + MEMORY.md (NON-NEGOTIABLE)
2. **MEMORY_CONTEXT.md → RECALL.md** renamed everywhere (daemon code, skill code, workspace files)
3. **Pre-Compaction Checkpoint Rule** added to AGENTS.md + MEMORY.md
4. **Post-Compaction Protocol** added to AGENTS.md + MEMORY.md
5. **Long-Running Work Protocol** added to AGENTS.md + MEMORY.md
6. Daemon restarted with updated code (new PID)

- **GA#1** (March 19): 64% retained, 36% lost. Agent didn't checkpoint during sprint. Justin had to manually re-teach everything. Pre-migration (no structured memory).
